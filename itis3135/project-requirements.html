Project Requirements and Design

Project Name: Virtual Art Gallery



Project Overview: This web application will be used to display multiple galleries of art and photgraphy simulating a physical art gallery. Made for storing and display of clients photos, with an intended audience of those who would like to see them, family or friends.

Client Information: Gavin Ferry, fellow UNCC student, aferry@uncc.edu

Site Map:


Home page -> ~3 Photo galleries -> Ext. Link to Image Source
Home Page -> Social Medias/ Contact Page

Home Page:

Display main info of page, and introduce client and content. Display Gallery Map that will lead to galleries
Viewers and those interested in the art are the intended audience
Button that shows image that is mapped to allow clicks on it that will lead to gallery pages
Has standard and main header with links to other pages, as well as social medias at bottom
Is dynamically styled to have accent colors change to random image that is diosplayed on load of page

Photo Gallery:


Viewers interested in the gallery content
image gallery that can be put as a slideshow for a sit back experience, as well and viewable in a row so that viewers can select the ones they llike to view in a larger format.
Has standard and main header with links to other pages, but is hidden during slideshow and large image viewing

Socials Page:
Viewers interested in the gallery content that would like to see other ways to view and interact with client
Has icons and links for social medias and other art sites that client would like to display
Has standard and main header with links to other pages
